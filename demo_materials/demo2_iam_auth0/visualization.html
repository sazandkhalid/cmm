Line 49 of cached_profile.md
[IAM Group Management / Auth Provider Abstraction Layer]

The get_iam_manager() factory function lacks an Auth0-specific case and falls through to the default Keycloak manager. This means when Auth0 is configured as the active authentication provider, IAM group management operations (create/delete/list) still use the Keycloak backend, causing Keycloak groups to appear in the IAM UI even when Auth0 is active. The provider abstraction is incomplete — Auth0 was never wired into the IAM manager dispatch logic.

Resolution
Add an Auth0 case to the get_iam_manager() factory and read AUTH_PROVIDER dynamically from os.environ at call time, not at module import time.

registry/utils/iam_manager.py AFTER (fixed) 
# Module-level constant removed entirely




# ...

# Line 607 — factory function
def get_iam_manager():
    provider = os.environ.get(  # dynamic!
        "AUTH_PROVIDER", "keycloak"
    ).lower()
    if provider == "keycloak":
        return KeycloakIAMManager()
    elif provider == "auth0":
        return Auth0IAMManager()
    return KeycloakIAMManager()
